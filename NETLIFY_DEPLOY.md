# Инструкция по деплою на Netlify

## Способ 1: Деплой через веб-интерфейс Netlify (Рекомендуется)

### Шаг 1: Создайте аккаунт на Netlify
1. Перейдите на https://www.netlify.com/
2. Нажмите "Sign up" и войдите через GitHub (самый простой способ)

### Шаг 2: Подключите репозиторий
1. В панели Netlify нажмите "Add new site" → "Import an existing project"
2. Выберите "Deploy with GitHub"
3. Авторизуйтесь через GitHub, если нужно
4. Выберите репозиторий: `NickitaStepin/testDeploy`

### Шаг 3: Настройте параметры сборки
Netlify автоматически определит настройки, но убедитесь что:
- **Build command:** `npm run build`
- **Publish directory:** `dist`
- **Branch to deploy:** `main`

Эти настройки уже указаны в файле `netlify.toml`, поэтому Netlify подхватит их автоматически.

### Шаг 4: Деплой
1. Нажмите "Deploy site"
2. Дождитесь завершения сборки (обычно 1-2 минуты)
3. Ваш сайт будет доступен по адресу вида: `https://random-name-12345.netlify.app`

### Шаг 5: Настройте кастомный домен (опционально)
1. В настройках сайта перейдите в "Domain settings"
2. Можно добавить свой домен или изменить имя сайта

## Способ 2: Деплой через Netlify CLI

### Шаг 1: Установите Netlify CLI
```powershell
npm install -g netlify-cli
```

### Шаг 2: Авторизуйтесь
```powershell
netlify login
```
Откроется браузер для авторизации.

### Шаг 3: Инициализируйте сайт
```powershell
netlify init
```
Следуйте инструкциям:
- Выберите "Create & configure a new site"
- Выберите команду (team)
- Назовите сайт или оставьте автоматическое имя

### Шаг 4: Деплой
```powershell
netlify deploy --prod
```

Или для предпросмотра:
```powershell
netlify deploy
```

## Способ 3: Drag & Drop (быстрый способ для теста)

1. Соберите проект локально:
   ```powershell
   npm run build
   ```

2. Перейдите на https://app.netlify.com/drop

3. Перетащите папку `dist` в окно браузера

4. Сайт будет задеплоен за несколько секунд

**Примечание:** Этот способ не подключён к GitHub, изменения нужно загружать вручную.

## Автоматический деплой

После подключения репозитория через Способ 1, Netlify будет автоматически деплоить сайт при каждом push в ветку `main`.

## Полезные команды Netlify CLI

```powershell
# Просмотр статуса
netlify status

# Открыть сайт в браузере
netlify open

# Просмотр логов
netlify logs

# Откатить к предыдущей версии
netlify rollback
```

## Решение проблем

### Проблема: Сборка падает
- Проверьте логи сборки в панели Netlify
- Убедитесь, что `package.json` содержит все необходимые зависимости
- Проверьте, что Node.js версия совместима (Netlify использует Node 18 по умолчанию)

### Проблема: Сайт показывает 404
- Убедитесь, что файл `netlify.toml` содержит redirect правила
- Проверьте, что `publish` директория указана как `dist`

### Проблема: Стили не загружаются
- Проверьте пути к файлам в `index.html`
- Убедитесь, что все ресурсы правильно импортированы
